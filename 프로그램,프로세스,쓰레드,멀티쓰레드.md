1. 프로그램 (Program): 하드디스크에 깔려 있는 게임, 카카오톡 같은 앱. (정적인 상태)

(예시: 식당 개업을 위한 '요리 레시피북')

2. 프로세스 (Process): 그 프로그램을 더블클릭해서 현재 실행 중인 상태, 메모리에 올라가서 돌아가고 있는 상태 (동적인 상태)

(예시: 오늘 영업을 시작한 '식당') 

3. 스레드 (Thread): 프로세스 내에서 실행되는 **가장 작은 작업의 단위(흐름)**

스레드는 프로세스(식당) 안에서 실제로 일을 하는 "작업자(직원)".

(예시: 모든 프로세스(식당)는 **최소 1개 이상의 스레드(메인 요리사)**를 가지고 있음.
이 요리사 혼자서 주문도 받고, 요리도 하고, 서빙도 한다면 그것을 **'싱글 스레드(Single-thread)'**라고 부름)


4. 멀티스레드 (Multi-thread): **하나의 프로그램 안에서 두 개 이상의 스레드가 동시에 작업을 수행하는 것**을 말합니다.

멀티스레드는 하나의 프로세스(식당) 안에 "작업자(직원)가 여러 명" 있는 상태

장점 : 혼자서 하던 일을 여러 명이 나누어 하거나 동시에 진행하므로 작업 속도가 훨씬 빠르고 효율적입니다.

(
  예시 1 : 웹 브라우저: 하나의 브라우저(프로세스) 안에서 스레드1은 파일을 다운로드하고, 스레드2는 유튜브 영상을 재생하고, 스레드3은 웹페이지를 스크롤합니다.

  예시 2 : 게임: 스레드1은 캐릭터의 움직임을 처리하고, 스레드2는 배경음악을 재생하고, 스레드3은 서버와 데이터를 주고받습니다.

  예시 3 : 메신저 (카카오톡): 스레드1은 친구에게 사진을 전송하고, 스레드2는 다른 친구가 보낸 메시지를 화면에 띄웁니다.
)
